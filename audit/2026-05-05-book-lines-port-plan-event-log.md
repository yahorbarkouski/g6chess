Date: 2026-05-05

## Scope

Investigate whether the book-lines UI/UX from the older G6 stack can be recreated in the current `g6explanation` frontend and backend, understand how it worked, and produce a comprehensive implementation plan.

No product code changes were made.

Key evidence:

- Old frontend `BookLinesView` accepted `bookLines`, `rootFen`, `onPreview`, and `activePreview`.
- Old frontend selected `currentBookLines` from `currentMarker.label_metadata.book_lines` only when `currentMarker.primary_class === "book"`.
- Old frontend rendered book lines instead of engine lines when book lines were present.
- Old backend `OpeningBookStatus` contained `is_book_move`, `book_lines`, `is_novelty`, `opening_name`, and `eco`.
- Old backend generated main book lines from the board after the played book move.
- Old backend payload assembly emitted `primary_class: "book"` and copied book data into `label_metadata`.
- Current frontend `MovePrimaryClass` already includes `book`.
- Current frontend API type does not include book metadata.
- Current backend `MoveQualityLabel` does not include `book`, and `GameMoveAnalysis` has no book metadata field.
- Current backend context contracts forbid extra fields.

## Hypotheses

| ID | Hypothesis | Status | Notes |
| --- | --- | --- | --- |
| H-001 | Book lines were supplied by the backend, not inferred in the frontend. | Confirmed | Old backend had a dedicated opening-book pass and payload metadata. |
| H-002 | Current frontend already has enough visual primitives for book moves. | Partially confirmed | Book class, label style, and move-list icon exist; the line view and mapping do not. |
| H-003 | Current backend can accept book fields without schema work. | Rejected | `ContextContract` and game-analysis contracts are strict; explicit fields are needed. |
| H-004 | The new backend should set `quality = "book"`. | Rejected | Current quality labels are engine/context semantics. Book status should be metadata plus frontend display override. |
| H-005 | Book-line preview should start from `fen_before`. | Rejected | Old frontend used `fen_after`, matching old backend continuations after the played book move. |
